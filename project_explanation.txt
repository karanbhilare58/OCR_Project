# 1. Project Overview

**Project Name:** AI Receipt OCR System

This project is an **end-to-end Optical Character Recognition (OCR) system** that extracts structured information from receipt images using **Computer Vision and Text Recognition techniques**.

The system processes receipt images through several steps:

1. Image preprocessing
2. Text detection
3. Optical Character Recognition (OCR)
4. Text cleaning
5. Data extraction using regex
6. Confidence scoring
7. Visualization through a Flask web interface

The final output extracts important receipt information such as:

* Date
* Subtotal
* Tax
* Total amount
* OCR confidence score

# 7. OCR Processing Pipeline

The system processes images through several steps.

## 7.1 Original Image

The receipt image uploaded by the user.

This is the raw input for the OCR system.

---

## 7.2 Grayscale Conversion

The image is converted to grayscale.

Why?

Color information is not necessary for text recognition.

Grayscale reduces image complexity and improves processing speed.

---

## 7.3 Thresholding

Thresholding converts the grayscale image into a binary image.

Pixels become either:

Black or White

This improves text visibility for OCR engines.

---

## 7.4 Text Region Detection

OpenCV is used to detect potential text regions in the image.

Bounding boxes are drawn around detected text.

These regions help visualize where OCR is extracting text.

---

## 7.5 OCR Recognition

The processed image is passed to **Tesseract OCR**.

Tesseract analyzes the image and converts the text into machine-readable characters.

Example output:

Date: 07/09/20
Subtotal: 3899.00
Tax: 341.16
Total: 4240.16

---

## 7.6 Text Cleaning

OCR output often contains noise such as:

* incorrect characters
* extra spaces
* broken words

The text cleaning module removes these issues.

---

## 7.7 Data Extraction

Regular expressions are used to extract key financial values.

Example patterns:

Date pattern
Subtotal pattern
Tax pattern
Total amount pattern

This converts raw OCR text into structured data.

---

## 7.8 Confidence Scoring

Tesseract provides a confidence score for each recognized word.

The system calculates an average confidence value.

This helps determine the reliability of the extracted data.

# 12. Interview Questions and Answers

## Q1: What is OCR?

OCR (Optical Character Recognition) is a technology that converts images containing text into machine-readable text.

---

## Q2: Why did you use Tesseract?

Tesseract is a powerful open-source OCR engine maintained by Google.

It supports multiple languages and integrates easily with Python.

---

## Q3: Why is preprocessing important in OCR?

Preprocessing improves OCR accuracy by:

* reducing noise
* enhancing contrast
* highlighting text regions

Without preprocessing OCR accuracy decreases significantly.

---

## Q4: Why convert the image to grayscale?

Grayscale simplifies the image by removing color information.

This reduces computational complexity and improves text detection.

---

## Q5: What is thresholding?

Thresholding converts a grayscale image into a binary image where pixels are either black or white.

This improves character visibility for OCR.

---

## Q6: Why use regex for data extraction?

OCR outputs unstructured text.

Regex allows us to extract specific patterns like:

dates
currency values
numbers

---

## Q7: What are bounding boxes?

Bounding boxes are rectangles drawn around detected text regions.

They visually show where OCR detected text.

---

## Q8: What is OCR confidence?

OCR confidence indicates how certain the OCR engine is about the recognized text.

Higher confidence means higher reliability.
